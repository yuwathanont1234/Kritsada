# แอปผู้พิทักษ์ครอบครัว — Core User Flow (v1)

จัดทำโดย ฟ้า — AI Solution UX/Flow

---

## 1. Persona ย่อ 3 บทบาท

**บทบาทที่ 1 — ผู้ดูแล (Caregiver)**
- อายุ 30-50 ปี วัยทำงาน
- เป้าหมาย: เห็นสถานะครอบครัวทั้งหมดในคลิกเดียว รับแจ้งเตือนทันทีเมื่อมีเหตุผิดปกติ
- ดิจิทัล: ใช้สมาร์ทโฟนคล่อง คุ้นเคย app สมัยใหม่
- Pain point: กังวลระหว่างวันทำงาน ไม่อยากเปิด app หลายตัว

**บทบาทที่ 2 — เด็ก**
- อายุ 6-15 ปี
- เป้าหมาย: ใช้ชีวิตปกติ มีปุ่ม SOS ฉุกเฉิน
- ดิจิทัล: คล่องสมาร์ทโฟน แต่ไม่ต้องการ config อะไรเอง
- Pain point: app ที่ทำให้รู้สึกถูก "สอดส่อง" มากเกินไป

**บทบาทที่ 3 — ผู้สูงอายุ**
- อายุ 60+ ปี
- เป้าหมาย: รู้สึกปลอดภัย ไม่รู้สึกเป็นภาระ
- ดิจิทัล: อ่อนแอ 80% มีปัญหาดิจิทัล คุ้นเคยแค่ LINE และโทรศัพท์
- Pain point: หน้าจอซับซ้อน ตัวอักษรเล็ก ไม่รู้ว่าต้องทำอะไร

## 2. User Flow หลัก

### A. Onboarding Flow

```
[ผู้ดูแล]
  |
  v
เปิด app ครั้งแรก
  |
  v
สมัคร/Login (Email หรือ Google)
  |
  v
สร้าง "ครอบครัว" — ตั้งชื่อ (เช่น "ครอบครัวสมชาย")
  |
  v
เลือก: "เพิ่มสมาชิก"
  |
  +---> [เพิ่มเด็ก] --> ส่งลิงก์ผ่าน LINE/SMS --> เด็กเปิดลิงก์ --> ยืนยัน 1 tap --> เริ่มใช้งาน
  |
  +---> [เพิ่มผู้สูงอายุ] --> ส่งลิงก์ผ่าน LINE --> ผู้สูงอายุเปิดลิงก์ (หรือผู้ดูแลตั้งให้แทน) --> ยืนยัน consent 1 tap --> เริ่มใช้งาน
  |
  v
ตั้งค่า geofence (บ้าน/โรงเรียน) — แผนที่ tap ครั้งเดียว
  |
  v
ตั้งค่าเตือนยา (วัน/เวลา) สำหรับผู้สูงอายุ
  |
  v
เข้าสู่ Dashboard หลัก
```

> จุดสำคัญ: ผู้สูงอายุที่ไม่คล่องดิจิทัล — ผู้ดูแลสามารถ "ตั้งค่าแทน" ได้โดยใช้โทรศัพท์ผู้สูงอายุ แล้วกด confirm แทน

---

### B. Daily Loop แต่ละบทบาท

**ผู้ดูแล — เช้าวันทำงาน**
```
เปิด app
  |
  v
Dashboard รวม (เห็น: เด็ก อยู่ที่ไหน | ผู้สูงอายุ เช็กอินวันนี้หรือยัง)
  |
  v
ทุกอย่างปกติ --> ปิด app ใช้ชีวิตต่อ
  |
  v
[ถ้ามี alert] --> notification pop ขึ้น --> กด action ทันทีจาก notification (ไม่ต้องเปิด app)
```

**ผู้สูงอายุ — เช้าทุกวัน**
```
รับ notification LINE / push "สวัสดีตอนเช้า กด 'ฉันปลอดภัย'"
  |
  v
เปิด app (หน้าจอมีแค่ปุ่มเดียวขนาดใหญ่)
  |
  v
กด "ฉันปลอดภัยวันนี้" (ปุ่มสีเขียวขนาดเต็มจอ)
  |
  v
หน้าจอแสดง "ดีมาก! ส่งข้อความถึงลูกแล้ว" --> จบ
```

**เด็ก — วันทำงาน/โรงเรียน**
```
เปิดแอปตามปกติ (ไม่ต้องทำอะไรเป็นพิเศษ)
  |
  v
แอปแชร์ตำแหน่งอัตโนมัติ (background)
  |
  v
[ฉุกเฉิน] กดปุ่ม SOS (เข้าถึงได้ 1 tap จากหน้าหลัก)
```

---

### C. Critical Alert Flow (สำคัญที่สุด)

```
TRIGGER (1 ใน 4 อย่าง):
  A. เด็กออกนอก geofence
  B. เด็กกด SOS
  C. ผู้สูงอายุตรวจจับล้ม (beta)
  D. ผู้สูงอายุไม่เช็กอิน ภายใน 9:00 น.
          |
          v
   ระบบส่ง Push Notification + LINE พร้อมกัน
   (ข้อความสั้น: "แม่ยังไม่เช็กอินวันนี้ — กดดูสถานะ")
          |
          v
   [ผู้ดูแลกด notification]
          |
          v
   หน้า Alert Detail:
     - แผนที่/สถานะล่าสุด
     - ปุ่ม "โทรหา [ชื่อ]" (1 tap)
     - ปุ่ม "แจ้งสมาชิกครอบครัวคนอื่น" (1 tap)
     - ปุ่ม "ฉันจัดการแล้ว" (ปิด alert)
          |
          v
   ผู้ดูแลกด action ที่เหมาะสม --> alert ถูกบันทึก --> กลับ dashboard
```

## 3. โครงสร้าง Navigation ฝั่งผู้ดูแล

Bottom Tab Bar (4 แท็บ) — mobile-first ไม่เกิน 4 แท็บ:

```
[Dashboard] [เด็ก] [ผู้สูงอายุ] [ตั้งค่า]
     |          |         |           |
 Overview    Map +     ยา +          สมาชิก
 ทั้งหมด    Geofence  เช็กอิน +     แจ้งเตือน
            SOS log   Fall log       consent
```

**Dashboard** = หน้าหลักรวม 2 โมดูล แบ่ง card แนวตั้ง:
- Card บน: สถานะเด็กทุกคน (ตำแหน่ง + geofence status)
- Card ล่าง: สถานะผู้สูงอายุทุกคน (เช็กอินหรือยัง + ยามื้อถัดไป)
- Alert banner ด้านบนสุด (ถ้ามี) — สีแดง ข้อความสั้น

## 5. จุดตัดสินใจ / Edge Case ที่ดีไซน์ต้องระวัง

**A. ผู้สูงอายุลืมเช็กอิน vs. เกิดเหตุจริง**

ระบบห้าม alert ทันที ต้องมี grace period:
```
9:00 น. ยังไม่เช็กอิน
  --> ส่ง reminder อีกครั้ง (LINE)
10:00 น. ยังไม่เช็กอิน
  --> ส่ง "soft alert" ให้ผู้ดูแล (ไม่ใช่สัญญาณฉุกเฉิน)
10:30 น. ยังไม่เช็กอิน
  --> ส่ง "urgent alert" + โทรสายตรง
```
ดีไซน์ต้องแสดงระดับของ alert ให้ชัด ไม่ให้ผู้ดูแลตื่นตระหนกทุกครั้ง

**B. Consent และความเป็นส่วนตัว**
- ผู้สูงอายุต้อง confirm ด้วยตัวเองก่อน (แม้ผู้ดูแลตั้งแทน) — UI ต้องมีขั้นตอน "ผู้สูงอายุยืนยัน" แยกออกมาชัดเจน
- เด็กอายุต่ำกว่า 13 ปี consent ต้องมาจากผู้ปกครอง — ระบุใน onboarding
- ผู้สูงอายุหรือเด็กสามารถ "ปิดแชร์ตำแหน่งชั่วคราว" ได้ แต่ผู้ดูแลได้รับแจ้ง

**C. แบตหมด / ออฟไลน์**
- ถ้าอุปกรณ์เด็กออฟไลน์เกิน 30 นาที → แจ้งผู้ดูแลว่า "ไม่พบสัญญาณ" (ไม่ใช่ alert ฉุกเฉิน)
- Dashboard ต้องแสดง timestamp ล่าสุดของข้อมูลเสมอ เช่น "ข้อมูลเมื่อ 14 นาทีที่แล้ว"
- ห้ามแสดงข้อมูลเก่าราวกับเป็นข้อมูลใหม่ — เป็น UX ที่อันตรายมาก

**D. Multi-caregiver (พ่อและแม่ดูแลพร้อมกัน)**
- ถ้า alert ถูก "รับ" โดยผู้ดูแลคนหนึ่งแล้ว ต้องซิงก์สถานะให้อีกคนเห็นทันที
- หน้า Alert Detail ต้องแสดงว่า "ลุงสมชายกำลังจัดการอยู่" เพื่อป้องกัน duplicate action

**E. Elder UX — ความสับสนเรื่อง "ทำอะไรไปแล้ว"**
- หลังกดเช็กอิน ต้องมี feedback ชัดเจน 100% (สีเขียว + ข้อความ + เสียง optional)
- ห้าม dismiss หน้าจอทันที ให้แสดงผล confirm ค้างไว้อย่างน้อย 3 วินาที

## สรุปหลักการออกแบบสำหรับ mook-ui-ux-designer

1. Elder screen = 1 หน้า 1 action ไม่มีทางเลือก
2. Alert ต้องมีระดับความรุนแรงชัดเจน (สี + ไอคอน + ข้อความ) ไม่ใช่แค่ notification สีเดียว
3. Dashboard ผู้ดูแลต้องอ่านได้ใน 3 วินาที โดยไม่ต้อง scroll
4. Timestamp บนข้อมูลทุกชิ้นที่มาจาก real-time
